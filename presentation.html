🚀 Server Alert Manager

Gmail Integration with Python & Bash

A robust, agentless monitoring solution for Linux servers.

🎯 Project Objective

To monitor critical server resources such as Disk Space and Free RAM.

When thresholds are crossed, the system instantly notifies the SysAdmin via an automated Gmail Alert.

🧠 Architecture (Separation of Concerns)

  • 1. The Monitor (Bash): Uses native, zero-overhead OS commands (df, free) to check metrics.
  • 2. The Sender (Python): Handles complex networking (SMTP SSL) to securely transmit the alert.

⚖️ Technical Decisions

Why did we reject Postfix / Sendmail?

  • Running an MTA consumes heavy server resources.
  • Requires complex DNS configs (SPF/DKIM) to avoid the Spam folder.
  • Risk of opening an Open Relay Vulnerability.

Our Solution:

Using Python smtplib acts as an authorized client connecting directly to Google's trusted infrastructure.

🔒 Security & Configuration

  • .env File: Credentials (like Google App Passwords) are completely isolated from the source code. No hardcoding!
  • monitor.conf: Alert thresholds (e.g. 80% Disk) are stored in a dedicated config file for high maintainability.
  • App Passwords: Adhering to Google's strict modern security policies by dropping Basic Authentication.
