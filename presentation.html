Server Alert Manager

Zero-Dependency Monitoring & Gmail Integration

GNU Bash 5.0+ Python 3.x SSL/TLS Secured

"Separation of concerns at the operating system level."

🛠️ Architectural Decisions

❌ The Old Way (Postfix/MTA)

  • Heavy on server resources (RAM/CPU).
  • Requires complex DNS setup (SPF, DKIM, DMARC).
  • Huge risk of becoming an Open Relay for hackers.
  • High probability of emails hitting the Spam folder.

✅ Our Solution (Python SMTP)

  • Zero background processes (Agentless).
  • Acts as a secure Client, not a Server.
  • Uses Google's robust infrastructure via Port 465 (SSL).
  • 100% Inbox delivery rate.

🧠 System Flow

Phase 1: Metric Collection ➡️ Bash scripts parse OS kernels natively.

Phase 2: Evaluation ➡️ Bash compares values against monitor.conf thresholds.

Phase 3: Execution ➡️ Python securely encrypts the payload and delivers it via Gmail.

🔍 Deep Dive: The Monitor

Why use awk and sed? Because they execute in O(N) time with ~0% CPU overhead.


# Extracting EXACT Disk Usage integer from OS
CURRENT_DISK_USAGE=$(df / | grep / | awk '{ print $5}' | sed 's/%//g')

# Extracting EXACT Free RAM in MBs
FREE_RAM_MB=$(free -m | awk 'NR==2{print $4}')

# The Logic Gate
if [ "$CURRENT_DISK_USAGE" -gt "$MAX_DISK_USAGE" ]; then
    python3 "$ALERT_SCRIPT" "$MESSAGE"
fi
Proof of Concept: If df outputs "85%", awk targets the 5th column, and sed sanitizes the string by stripping "%", returning purely 85 for safe integer comparison.

🔒 Deep Dive: The Transmitter

Utilizing Python's smtplib for secure TLS/SSL handshakes.


from email.message import EmailMessage
import smtplib, os

# Using modern EmailMessage class to prevent Spam flagging
msg = EmailMessage()
msg.set_content(alert_payload)
msg['Subject'] = 'CRITICAL: Server Alert'

# Secure SSL connection on Port 465 (No STARTTLS required)
with smtplib.SMTP_SSL('smtp.gmail.com', 465) as smtp:
    smtp.login(os.environ.get('GMAIL'), os.environ.get('APP_PASS'))
    smtp.send_message(msg)

🛡️ Security Protocols Enforced

  • No Hardcoded Secrets: All credentials are obfuscated inside a .env file.
  • Google App Passwords: Basic Authentication is deprecated by Google. We generated a scoped App Password. If leaked, it only grants email sending rights, protecting the main Google account.
  • Configuration Management: Thresholds (80% disk, 500MB ram) are isolated in monitor.conf. System behavior can be modified by ops teams without touching the source code.
